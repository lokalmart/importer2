LOKALMART IMPORTER STUDIO - PANDUAN CHATGPT MEMBUAT XLSX SIAP IMPORT
Versi: v0.3.4 Studio Safe
Target: Odoo Online / Odoo 18 / Vercel Lokalmart Importer Studio
Tujuan: agar ChatGPT selalu membuat XLSX yang bisa langsung diimport, cepat, minim warning, dan aman untuk import ulang.

====================================================================
A. PRINSIP DASAR
====================================================================

1. XLSX harus dibuat untuk mesin import, bukan untuk dibaca manusia saja.
2. Setiap sheet data harus memakai nama technical model Odoo, contoh:
   - ir.model.fields
   - res.partner
   - product.category
   - product.public.category
   - product.tag
   - product.attribute
   - product.attribute.value
   - product.template
   - product.supplierinfo
   - photo_import_queue
3. Sheet catatan boleh ada, tetapi jangan dianggap sheet data. Gunakan awalan underscore:
   - _README
   - _source_notes
   - _fix_notes
   - _validation
4. Workbook multi-sheet wajib punya sheet 00_import_order.
5. Setiap sheet data wajib punya kolom:
   - __action
   - _external_id
   - _model
6. Semua record wajib punya _external_id stabil, unik, dan tidak berubah saat import ulang.
7. Semua custom field Odoo Online wajib memakai prefix x_.
8. Jangan mencampur data inti dengan QWeb/website.page kecuali benar-benar dibutuhkan.

====================================================================
B. URUTAN IMPORT WAJIB
====================================================================

Gunakan sheet 00_import_order dengan kolom minimal:
- sequence
- sheet
- model
- phase
- note

Urutan aman umum:
1. ir.model
2. ir.model.fields
3. ir.model.access
4. master data induk:
   - res.partner.category
   - res.partner
   - product.category

   - product.attribute.value
5. product.template
6. data relasi:
   - product.template.attribute.line
   - product.supplierinfo
7. photo_import_queue
8. custom data lain jika ada

Catatan penting:
- Setelah ir.model.fields berhasil, Importer Studio harus refresh metadata Odoo sebelum memproses sheet berikutnya.
- Foto hanya boleh diproses setelah product.template berhasil created/updated.
- product.supplierinfo hanya boleh diproses setelah product.template dan res.partner target ada.

====================================================================
C. FORMAT KOLOM DASAR SETIAP SHEET DATA
====================================================================

Kolom wajib:
- __action
- _external_id
- _model

Nilai __action yang aman:
- upsert      : update jika external ID ada, create jika belum ada.
- create      : create saja. Gunakan terbatas.
- update      : update saja. Gunakan untuk patch.
- delete      : hindari kecuali patch khusus.

Contoh:
__action: upsert
_external_id: lokalmart.product_001_nasi_jamblang_paket_ayam
_model: product.template

Namespace external ID:
- Gunakan namespace konsisten: lokalmart.xxx
- Jangan memakai spasi.
- Jangan memakai karakter aneh.
- Gunakan lowercase dan underscore.

Contoh aman:
- lokalmart.partner_koperasi_lokal_kejaksan
- lokalmart.cat_lm_barang_dagang_sembako
- lokalmart.public_cat_kuliner_lokal_nasi_jamblang
- lokalmart.tag_h_plus_1
- lokalmart.product_001_nasi_jamblang_paket_ayam

====================================================================
D. ATURAN RELASI MANY2ONE DAN MANY2MANY
====================================================================

1. Many2one harus pakai suffix _external_id.
   Contoh:
   - categ_id_external_id
   - parent_id_external_id
   - partner_id_external_id
   - product_tmpl_id_external_id
   - x_lm_vendor_id_external_id

2. Many2many harus pakai suffix _external_ids.
   Isi beberapa external ID dipisahkan koma.
   Contoh:
   - public_categ_ids_external_ids
   - product_tag_ids_external_ids
   - category_id_external_ids

3. Jangan pakai ID database numerik Odoo.
   Salah:
   categ_id: 15
   Benar:
   categ_id_external_id: lokalmart.cat_lm_kuliner_siap_jual

4. Relasi external ID harus sudah ada dari sheet sebelumnya.
   Jika belum ada, importer akan memberi error:
   External ID relasi tidak ditemukan.

====================================================================
E. MODE SUPER CEPAT DAN ERROR BOOLEAN value.lower
====================================================================

Mode Super Cepat memakai method native-like Odoo load(fields, data).
Pada mode ini, Odoo membaca nilai seperti import wizard native.
Karena itu nilai boolean harus dikirim sebagai TEKS, bukan boolean JavaScript.

ERROR YANG WAJIB DICEGAH:
AttributeError: 'bool' object has no attribute 'lower'
atau:
value.lower is not a function

Penyebab:
- XLSX dibaca menjadi boolean JavaScript true/false.
- Lalu dikirim ke Odoo load().
- Converter boolean Odoo mengharapkan string dan memanggil value.lower().
- Karena value berupa bool, import gagal.

Aturan pembuatan XLSX:
- Untuk mode Super Cepat, semua boolean di sel XLSX harus berupa teks:
  TRUE atau FALSE
- Jangan isi sebagai boolean Excel asli TRUE/FALSE jika library akan membacanya sebagai bool.
- Lebih aman tulis sebagai string teks: "TRUE" dan "FALSE".
- Untuk sheet ir.model.fields, nilai required/readonly/store/index/copied juga pakai teks:
  TRUE / FALSE

Contoh aman:
website_published: FALSE
sale_ok: TRUE
purchase_ok: TRUE
required: FALSE
store: TRUE
copied: TRUE

Contoh rawan:
website_published: true
required: false

Pencegahan di Importer Studio v0.3.4:
- Saat mode Super Cepat, importer otomatis mengubah boolean JS menjadi teks TRUE/FALSE sebelum memanggil Odoo load().
- Walaupun begitu, ChatGPT tetap wajib membuat XLSX dengan nilai boolean teks untuk menghindari error lintas tool.

====================================================================
F. ATURAN TIPE NILAI DI XLSX
====================================================================

1. Boolean:
   Gunakan teks TRUE/FALSE.

2. Angka:
   Gunakan angka normal untuk harga dan cost:
   list_price: 15000
   standard_price: 9000
   Namun mode Super Cepat akan mengirimnya ke Odoo sebagai string agar Odoo load parsing sendiri.

3. Tanggal:
   Gunakan format ISO:
   YYYY-MM-DD
   Contoh: 2026-06-05

4. Datetime:
   Gunakan format:
   YYYY-MM-DD HH:MM:SS
   Hindari timezone lokal di sel XLSX kecuali field memang butuh.

5. Selection:
   Isi dengan technical value, bukan label tampilan jika diketahui.
   Contoh product.type Odoo 18 bisa berbeda tergantung database.
   Jika ragu, hindari field selection rawan dan pakai field default Odoo saja.

6. HTML:
   Untuk deskripsi HTML pendek boleh pakai description_sale atau website_description.
   Jangan memasukkan JavaScript/QWeb di sheet produk.

Changing the type of a field is not yet supported. Please drop it and create it again!

Solusi:
- Importer Studio akan skip field existing dengan tipe berbeda.
- ChatGPT wajib memilih nama field baru jika diketahui field lama konflik.

- _model

- image_url harus URL publik tanpa login.
- Jangan pakai Google Drive private, WhatsApp, marketplace private, atau URL yang butuh cookie.
- Foto diproses setelah product.template created/updated > 0.

Error yang dicegah:
External ID record tidak ditemukan: lokalmart.photo_queue_xxx

Penyebab:
- record_external_id keliru diisi ID queue, bukan ID produk.

Benar:

Salah:

R. ATURAN PERFORMA SUPER CEPAT
====================================================================

Agar impor cepat seperti native Odoo:
1. Gunakan mode Super Cepat untuk data besar.
2. Kurangi warning. Warning berulang memperlambat proses.
3. Jangan kirim kolom yang tidak ada di model target.
4. Pecah import menjadi fase:
   - struktur/field
   - master data
   - produk
   - relasi
   - foto
5. Gunakan batch besar untuk data bersih:
   - master data: 80-200 row
   - product.template: 50-150 row
   - foto: 5-15 row karena download gambar berat
6. Untuk ir.model.fields tetap gunakan handler khusus, bukan load native biasa.
7. Untuk row yang rawan relasi/foto, gunakan mode Aman jika perlu debugging.

====================================================================
T. DAFTAR ERROR DAN CARA MENCEGAH
====================================================================

1. Error:
AttributeError: 'bool' object has no attribute 'lower'
Penyebab:
Boolean JS dikirim ke Odoo load().
Pencegahan:
Gunakan TRUE/FALSE sebagai teks. Importer Studio juga convert otomatis di mode Super Cepat.

2. Error:
Changing the type of a field is not yet supported.
Penyebab:
Mencoba membuat field dengan nama yang sama tetapi ttype berbeda.
Pencegahan:
Jangan ubah tipe field lama. Buat nama field baru, contoh x_lm_delivery_label.

3. Error:
ir.model.fields row wajib punya model dan name.
Penyebab:
Sheet ir.model.fields tidak punya kolom model/name.
Pencegahan:
Selalu isi model dan name di setiap row ir.model.fields.

4. Error:
External ID relasi tidak ditemukan.
Penyebab:
Relasi mengarah ke record yang belum dibuat atau external ID salah.
Pencegahan:
Perbaiki 00_import_order dan external ID relasi.

5. Error:
External ID record tidak ditemukan pada photo_import_queue.
Penyebab:
record_external_id mengarah ke queue ID, bukan produk.
Pencegahan:
record_external_id harus external ID product.template.

6. Warning:
Kolom active dilewati karena field tidak ada di product.category.
Pencegahan:
Hapus active dari product.category jika target Odoo tidak punya field itu.

7. Error:
You are not allowed to create custom model records.
Penyebab:
Custom model x_* belum punya ACL.
Pencegahan:
Buat ir.model.access sebelum sheet data custom model.

8. Error:
Barcode(s) already assigned.
Penyebab:
Barcode duplikat.
Pencegahan:
Kosongkan barcode jika tidak yakin unik.

9. Error:
Wrong value for selection field.
Penyebab:
Mengisi label tampilan, bukan technical value.
Pencegahan:
Gunakan technical value atau hindari field selection rawan.

10. Timeout / FUNCTION_INVOCATION_TIMEOUT.
Penyebab:
Batch terlalu besar, terlalu banyak warning, foto terlalu berat, atau server lambat.
Pencegahan:
Kecilkan batch, bersihkan kolom invalid, pisahkan foto, gunakan retry.

====================================================================
X. INSTRUKSI UNTUK CHATGPT SETIAP MEMBUAT XLSX LOKALMART
====================================================================

Saat user meminta file XLSX untuk Lokalmart Importer Studio, ChatGPT wajib:
1. Buat workbook dengan 00_import_order.
2. Buat sheet data memakai nama technical model.
3. Gunakan _external_id stabil.
4. Import schema/field dulu jika butuh custom field.
5. Hindari kolom yang tidak ada di target model.
6. Tulis boolean sebagai teks TRUE/FALSE.
7. Buat foto lewat photo_import_queue.
8. Pastikan record_external_id foto mengarah ke produk.
9. Set website_published FALSE kecuali user minta publish.
10. Tambahkan sheet _validation berisi ringkasan jumlah row, jumlah external ID, dan catatan risiko.
11. Jangan memberi file jika masih ada relasi ke external ID yang belum dibuat di file atau belum diasumsikan ada di database.
